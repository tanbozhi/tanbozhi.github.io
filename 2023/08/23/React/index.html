context上下文

当某个组件创建上下文后，上下文的数据会被所有后代组件共享

一般情况下，用于第三方组件（通用组件）

旧版：
创建上下文

只有类组件才可以创建上下文

必须约束上下文数据的类型

必须写得到上下文中的数据的方法getChildContext,该方法会在render方法之后运行

使用上下文的数据

使用的组件必须有contextTypes静态属性

上下文的数据修改

上下文的数据来自于状态或者属性，修改上下文数据，只能修改状态和属性

子组件修改上下文数据的方法，通过上下文中的一个函数(自定义)来修改

新版
创建上下文

通过React.createContext()创建

返回的是一个有Provider,Consumer属性的对象

Provider：生产者，是一个组件，该组件会创建一个上下文，通过value属性赋值

Consumer ：消费者，是用来在函数组件中获取数据的

上下文可以提到一个单独文件写

使用上下文的数据

必须要有静态属性 contextType，contextType应赋值为创建的上下文对象

上下文的数据修改

上下文的数据来自于状态或者属性，修改上下文数据，只能修改状态和属性

子组件修改上下文数据的方法，通过上下文中的一个函数(自定义)来修改

PureComponent纯组件

用于避免不必要的渲染(运行render函数)，从而提高效率

如果一个组件的属性和状态都没有发生变化，就不会重新渲染该组件

原理是在shouldComponentUpdate中对比属性和状态有没有发生变化，比较是浅比较

Memo组件

传入一个函数组件，它会返回一个继承了PureComponent的类组件，从而让函数组件也能实现PureComponent的功能

render props

有时候，某些组件的各功能及其处理逻辑几乎相同，只是显示的界面不一样，可以用render props或者hoc来解决这一问题

render props

Portals

将一个react元素渲染到指定的元素中

注意：

由于react中的事件是包装过的，它的事件冒泡是根据虚拟dom树来冒泡的，于真实dom树无关

错误边界

默认情况下，若一个组件在渲染期间发生错误，会导致整个组件树全部被卸载

错误边界是一个组件，该组件可以捕获到渲染期间子组件发生的错误，并有能力阻止错误继续传播

  1. 编写生命周期函数getDerivedStateFromError

    静态函数

    运行时间点：渲染子组件过程中，发生错误后，在更新页面之前

    只有子组件发生错误后，才会运行该函数

    该函数返回一个对象，react会将该对象的属性覆盖掉当前组件的state

    通常，该函数用于改变状态

    参数：错误对象

  2. 编写生命周期函数componentDidCatch

    实例方法

    运行时间点：渲染子组件过程中，发生错误后，更新页面之后

    由于其运行事件点比较靠后，因此不太会在该函数中改变状态

    通常，该函数用于记录错误消息

    注意：

自身的错误不会被捕获

异步的错误不会被捕获

事件中的错误不会被捕获

HOOK

React 16.8.0之后才出现hook

专门用于增强函数组件的功能（只能在函数组件中使用）

一个函数中可以有多个状态

常用： useState useEffect

useState

作用：充当函数组件的状态

该函数返回值是一个数组，有两个参数，第一个参数是状态的值，第二项是改变这个状态的函数

原理

当组件节点被创建的时候，会生成一个状态表格，他会检查这个表格中有没有东西，没有就将默认状态加到表格中，有的话就直接使用状态值

注意点

useState不能出现在代码块中

useState返回的函数（第二项）引用不变

useState改变数据后，如果数据不变(object.is比较)，就不会导致重新渲染

useState改变数据，传入的值会直接覆盖之前的数据，不会进行合并

useState改变数据如果是异步的，会将多个状态变化合并后再渲染，所以不能被信任之前的状态，连续改变状态最好使用回调形式修改
